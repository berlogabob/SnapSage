### 1. Концепция и описание проекта

**SnapSage** — это личное кросс-платформенное приложение, которое автоматически превращает все ваши скриншоты (с телефона, компьютера, облачных дисков) в умную, полностью поисковую базу знаний.

**Что происходит автоматически после установки:**
1. Вы делаете скриншот (на любом устройстве).
2. Приложение находит его в выбранных папках или облаке.
3. Модель ИИ (Ollama) выполняет точное распознавание текста + понимает визуальный контекст («чёрный кожаный рюкзак Peak Design Everyday Backpack 20L за 12 990 ₽ на модели в городском стиле»).
4. Текст переводится на все выбранные вами языки (русский, английский, португальский и любые другие).
5. Если на скриншоте событие (концерт, встреча, билет, запись к врачу) — оно автоматически добавляется в Google Calendar и создаёт страницу/блок/Daily Note в ваших заметках (Logseq, Obsidian, Notion или Bear).
6. Присваиваются умные теги.
7. Всё сохраняется в локальной базе, и вы можете мгновенно искать по любому слову, бренду, дате или описанию.

**Главные принципы:**
- 100 % приватность и оффлайн-режим по умолчанию (ни один скриншот не покидает ваше устройство).
- Максимально простой онбординг для новичков и пожилых людей.
- Работает на всех платформах: Android, iOS, macOS, Windows, Linux (как один файл .AppImage).
- Полная интеграция с вашими заметками (Logseq — приоритет).
- Автоматическое тегирование + возможность ручной правки.
- Тёмная тема по умолчанию, крупный шрифт, высокий контраст.

Приложение решает реальную боль современного человека: «Я точно где-то видел эту информацию, но где именно?»

### 3. User Journey (полный путь пользователя)

**День 1 — Первый запуск**
1. Устанавливаете приложение (apk / ipa / dmg / exe / AppImage).
2. Открываете → приветственный экран с коротким видео.
3. Проходите 5-экранный онбординг (выбор источников скриншотов, языков, интеграций).
4. Подключаете Google Calendar (одна кнопка).
5. Приложение скачивает модели Ollama (один раз, ~4–8 ГБ).
6. Готово. Теперь просто делайте скриншоты как обычно.

**Ежедневное использование**
1. Делаете скриншот на телефоне или компьютере.
2. Через 10–60 секунд в приложении появляется уведомление «Скриншот обработан».
3. Открываете приложение → видите карточку с текстом, описанием, переводами и тегами.
4. Если это концерт — нажимаете «Добавить в календарь» (или оно уже добавлено автоматически).
5. Если нужно — открываете детали и правите теги или отправляете в Logseq.

**Поиск**
1. Открываете приложение.
2. Вводите «рюкзак peak design» или «концерт март».
3. Мгновенно видите все релевантные скриншоты.

**Настройки**
В любой момент меняете языки, включаете/выключаете авто-удаление, меняете модель Ollama и т.д.

### 4. Все экраны приложения (подробное описание дизайна и наполнения)

**Экран 1. Splash Screen**  
- Тёмный фон.  
- Большая иконка приложения в центре с анимацией загрузки (пульсирующий круг).  
- Текст снизу: «SnapSage • Загружаем модели ИИ…»

**Экран 2–6. Онбординг (5 экранов, PageView с индикатором)**  
Экран 2: «Откуда брать скриншоты?»  
- Заголовок крупным шрифтом.  
- 6 больших карточек с иконками:  
  - Яндекс Диск (с пометкой «включая папку Снимки экрана»)  
  - Google Drive  
  - Рабочий стол / Downloads / Pictures (автоопределение)  
  - Галерея телефона  
  - Любая папка  
  - Nextcloud / Syncthing  
- Кнопка «Далее».

Экран 3: «На каких языках работать?»  
- Поисковая строка сверху.  
- Список чекбоксов (по 2 в ряд): Русский, Английский, Португальский (Бразилия), Португальский (Европа), Немецкий, Французский, Испанский и т.д. (всего ~30 языков).  
- Кнопка «Выбрать все популярные».

Экран 4: «Интеграции с заметками»  
- Галочки: Logseq, Obsidian, Notion, Bear.  
- Под каждой галочкой короткое описание («Автоматически создаст Daily Notes и linked blocks»).

Экран 5: «Дополнительные настройки»  
- Галочка «Автоматически удалять обработанные скриншоты»  
- Галочка «Автоматически добавлять события в Google Calendar»  
- Кнопка «Подключить Google Calendar».

Экран 6: «Готово!»  
- Большая зелёная кнопка «Начать работу».  
- Короткое видео-туториал (15 сек).

**Главный экран (Home Tab)**  
- AppBar с логотипом и поиском.  
- Раздел «Последние скриншоты» — горизонтальный список карточек (миниатюра + заголовок + 2 тега).  
- Раздел «Сегодняшние события» — карточки концертов/встреч.  
- BottomNavigationBar: Домой | Поиск | События | Настройки.

**Экран Поиска**  
- Большая поисковая строка вверху.  
- Фильтры: По дате, По тегам, По языку.  
- Результаты в виде сетки или списка карточек.

**Экран Детали скриншота**  
- Большая миниатюра сверху (нажатие — открывает оригинал).  
- Вкладки (TabBar):  
  1. OCR Content (чистый Markdown)  
  2. Visual Description  
  3. Переводы (по языкам)  
  4. Теги (список чипов, можно добавлять/удалять)  
- Кнопки действий снизу: Открыть в Logseq, Добавить в календарь, Экспорт .md, Удалить.

**Экран Событий**  
- Список карточек событий с датой, временем, местом и кнопкой «Открыть в календаре».

**Экран Настроек**  
- Разделы:  
  - Языки (изменить)  
  - Источники скриншотов  
  - Интеграции заметок (вкл/выкл)  
  - Модель ИИ (qwen3.5:2b / glm-ocr)  
  - Авто-удаление  
  - О приложении, Экспорт данных, Резервное копирование

**Дизайн-система**  
- Цветовая схема: тёмная по умолчанию (#121212 фон, #FFFFFF текст).  
- Минимальный размер текста 18 pt.  
- Все кнопки высотой минимум 56 dp.  
- Material 3 дизайн с адаптацией под iOS.

### 5. Технический стек (2026)

- Flutter 3.29+ (один код для всех платформ)  
- Dart 3.7+  
- State management: Riverpod 2.5  
- База данных: drift (SQLite)  
- Ollama: через http-пакет + structured JSON output  
- Модели: qwen3.5:2b (основная), glm-ocr (для таблиц)  
- Эмбеддинги: sentence-transformers all-MiniLM-L6-v2  
- Синхронизация: dart:io + isolates + официальные SDK Яндекс Диск / Google Drive  
- Интеграции заметок: Logseq Graph API, Obsidian Local REST, Notion API, Bear API  
- Сборка Linux: flutter_distributor → .AppImage
